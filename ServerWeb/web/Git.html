Anche Git ha giocato

Git è un sistema software di controllo di versione distribuito, creato da Linus Torvalds nel 2005. È ben presto diventato un sistema di controllo di versione, direttamente utilizzabile da riga di comando; vari progetti software usano Git per tale attività, tra cui il kernel Linux.


Caratteristiche

  • Grande facilità di branch
  • Git dà a ogni sviluppatore una copia locale dell'intera cronologia di sviluppo. Le modifiche vengono copiate da un repository a un altro.
  • È semplice utilizzare HTTP e FTP per pubblicare i repository.
  • Esiste la possiblità di emulare gli strumenti CVS.
  • Gestione efficiente di grandi progetti. Git è molto veloce e scalabile

Considerazioni

Grazie alla possibilità di effettuare commit in locale (nella sandbox della repository presente direttamente sul sistema in utilizzo), Git promette grande velocità nel tenere traccia del proprio lavoro.
Ciò che un sistema del genere non incoraggia è il frequente utilizzo della funzione "push", che è quella che effettivamente porta le modifiche effettuate al codice nella repository "sulla nuvola", visibli a tutti gli altri sviluppatori del team. La condivisione del codice ne risulta inevitabilmente compromessa, difatti un programmatore è incentivato a mantenere il lavoro in locale il più a lungo possibile, con conseguenti rischi di pericolosi merge e (forse peggio) rischi di "botte di codice improvvise", che rendono difficile seguire passo-passo lo sviluppo del codice altrui.
